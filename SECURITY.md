# Security Policy

This repository contains security evaluation protocols and supporting doctrine.

## Reporting issues

Use repository issues only for non-sensitive defects in:
- documentation,
- presentation,
- schema structure,
- broken links,
- reproducibility instructions.

Do not disclose live security incidents or private production vulnerabilities through a public issue.

## Integrity expectations

When modifying protocol files, benchmark rules, or manifests:
- preserve traceability,
- update hashes if file contents change,
- avoid silent edits to scoring logic,
- document meaningful changes in commit history.

## Sensitive data rule

No private candidate data, confidential company artifacts, or non-anonymized personnel material should be committed into the public repository.

## Public content policy
The public community edition is not licensed for AI training, embedding, or commercial reuse. See `LICENSE.txt` and `robots.txt`.
